2. About¶

I am trying to predict home energy use based on previous energy use. My inputs are:

  • Hourly Energy Use (kW hrs)
  • Month value (1-12)/12
  • Type of home (integer value 1-6)

I decided on a time window of 168 hours previous to the time of interest (look back one week and include all of those values with the current observation).

1) I initially created this model using only one house's data. This worked well and I could get a MSE of 0.34 on the test data with very reasonable-looking predictions.

2) Then I expanded my input data to include all 16 houses in the dataset that had less than 1% of their records missing and were in the same city (Vancouver). The list of these houses is at the bottom of this cell. I used panda's interpolation function to impute the missing data and combined all the data into one large training set (~10x more observations than the original single house's dataset). I also switched to Mean Absolute Error as I reasoned that with a larger dataset I would have more outliers and MAE should be more robust to that than MSE.

The model is an RNN with LSTM and I am training on the first 70% of the data from each home (combined) and validating on the remaining 30% from each home (combined).

def get_df(houselist,housetype,n):
    print("House Num",houselist[n])
    fname = "data/Residential_"+str(houselist[n])+".csv"
    singleDF = pd.read_csv(fname)
    singleDF["date"]=pd.to_datetime(singleDF["date"])
    singleDF["tstamp"]=singleDF.apply(lambda x: x.date+pd.Timedelta(hours=x.hour), axis=1)
    singleDF.drop(["date","hour"], axis=1, inplace=True)
    singleDF.set_index(["tstamp"], inplace=True)
    print("Using linear interpolation to impute data for",len(singleDF[singleDF["energy_kWh"].isna()]),"records")
    singleDF = singleDF.assign(energy_kWh=singleDF["energy_kWh"].interpolate(method="linear"))
    print("Resulting Missing Data Count:",len(singleDF[singleDF["energy_kWh"].isna()]))
    singleDF["houseType"]=housetype[n]
    return(singleDF)

def split_data(df, split_percent=0.7):
    n = len(df)
    tt_split = int(n * split_percent)
    #splitTest = n - rangeTrain

    train_data = df.iloc[0:tt_split]
    test_data = df.iloc[tt_split:n]
    print(n, "records total")
    print("Training Set: ",len(train_data), "\nTest Set: ",len(test_data))
    return (train_data,test_data)

def create_timewindows(df, response_series, steps=1):
    pred_list1 = []
    pred_list2 = []
    pred_list3 = []
    response_list = []
    for i in range(len(df) - steps):
        pred_list1.append(df.iloc[i:(i + steps)].values)
        pred_list2.append(df.iloc[i:(i + steps)].index.month.values/12.0)
        response_list.append(response_series.iloc[i +steps])
    
    pred_array1 = np.array(pred_list1)
    pred_array2 = np.array(pred_list2)

About the Model¶

I started with a simple model and added depth and more units until the training stopped working (i.e. could not reduce loss and just settled on predicting the mean of all energy use as the output). Using the adam optimizer with default learning rate gave good results and fiddling with the learning rate did not improve the performance.

Results - Discussion¶

Combining the energy use data for all the relevent houses worked out pretty well. The predicted trends look very reasonable. However, unlike the original work I did with only one house's energy use data, this predicted values here fall below the minimum of the actual values. This was worse in earlier generations of the model that hadn't been trained as long but it is still present even after ~50 epocks. I suspect this is because different houses have different minimum energy use. It is possible that further training could fix this (and increase some of the peaks to match the actual peaks).

Main takeaways:

  • LSTM RNN's are surprisingly accurate on time series data and seem to require less initial investigation than ARIMA, no need try to estimate and remove seasonality, etc. This seems like a very useful tool to have available when prediction accuracy is the goal.
